What is DGFacade?

DGFacade is a high-performance data gateway facade built for enterprise-scale request processing. Written in Java with Spring Boot, it provides a unified facade over pluggable handler chains, message broker integrations, and multi-protocol ingestion pipelines.

Instead of building separate services for each processing need, DGFacade routes requests to configurable handler chains — the compute-to-data paradigm. Handlers can be written in Java, Scala, or Python.

High Performance

Sub-millisecond handler dispatch with reactive streams, async processing, and connection pooling across all broker integrations.

Handler Chaining

Compose complex processing pipelines from simple handlers. Each handler transforms the request before passing to the next in the chain.

Python Handler Support

Write handlers in Python with the same start/compute/stop contract. Py4J worker pool with auto-restart and admin UI. NEW in 1.6.1

Multi-Protocol Ingestion

Ingest data from Kafka, RabbitMQ, ActiveMQ, or REST endpoints. Configurable input/output channels with routing rules.

Real-Time Monitoring

Sparkline metrics, live log streaming, Prometheus endpoints, health checks, and 18 admin dashboard pages.

Enterprise Security

Role-based access control, API key management, Spring Security integration, and audit logging for all admin operations.

Version History

  • Health Monitoring: Periodic health checks with configurable thresholds and automatic recovery
