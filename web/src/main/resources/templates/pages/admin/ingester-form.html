Configuration Tips
Three-Layer Resolution

Ingesters use a 3-layer config chain: Broker (connection) → Input Channel (destinations) → Ingester (overrides). Each layer can override values from the previous one.

Input Channel

The input channel must be created first under Admin → Input Channels. It defines the channel type (Kafka, ActiveMQ, etc.) and references a broker for connection details.

Overrides

Use overrides to customize per-ingester settings without modifying the shared channel/broker configs. Common overrides include group_id, max_poll_records, and auto_offset_reset.

Naming Convention

Use descriptive names like kafka-requests, activemq-orders, or filesystem-batch. The ID becomes the JSON config filename.

Config files are saved to config/ingesters/<id>.json and can also be edited directly on disk.
