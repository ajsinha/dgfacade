Configuration Tips
Broker ID Naming

Use a descriptive name like prod-kafka, dev-activemq, or staging-rabbitmq-ssl. This becomes the JSON filename.

Environment Variables

Use ${ENV_VAR} syntax for sensitive values like passwords. Example: ${KAFKA_PASSWORD} will be resolved at runtime.

SSL Certificates

Place certificate files in config/certs/<broker>/. PEM format is recommended for modern deployments. JKS is available for legacy Java-based systems.

Kafka SSL

For Kafka mTLS, enable SSL and set all three PEM paths. Add security.protocol=SSL in Additional Properties.

ActiveMQ Failover

For HA, use failover URI: failover:(ssl://host1:61617,ssl://host2:61617)?maxReconnectDelay=5000

IBM MQ

The SVRCONN channel must be configured on the queue manager to allow client connections. Default dev channel: DEV.APP.SVRCONN

Config files are saved to config/brokers/<id>.json and can also be edited directly on disk.
